# Documentation Guide

## Purpose

This folder is the permanent documentation base for ForjaDev Marketplace.

The documents here are not optional background notes. They are operational inputs for analysis, implementation, QA and architectural decisions.

Use these documents as the current documentation layer for the real project state:

## How to use this folder

- Check relevant docs before proposing or implementing changes.
- If documents conflict, stop and surface the conflict before continuing.
- If architecture, routes, behavior or verification changes, update the affected docs.
- Prefer operational truth over historical planning assumptions.
